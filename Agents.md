# Project Agents Rules

1. 这是一个用于测试功能的 UE 项目。
2. 该项目是纯 C++ / 蓝图项目，不使用任何脚本语言，包括 Lua。
3. 所有正式功能都必须封装在插件中实现。
4. `Empty_54/Source` 中不放功能代码，最多只保留测试代码、壳代码或必须的工程入口代码。
5. UE 源码目录固定为 `S:\UnrealEngine\UE_5.4\Engine`，后续涉及 UE 代码问答、源码定位或引擎相关编写时，默认以该目录为准。

## 变更记录

1. 所有功能相关的修改和讨论记录，必须写入对应插件根目录下的 `UpdateLog`。
2. `UpdateLog` 可以按日期创建新文件。
3. 每个记录文件开头必须标记本次修改内容，方便后续检索。
4. 所有模块相关的阅读记录、讨论记录、思考记录，都必须在对应模块目录中进行增删改，不能集中放到项目根目录或无关目录。
5. 例如 HTN 模块目录为 `Plugins/AI/HTN`，HTN 相关的阅读记录、讨论记录、思考记录都必须写入 `Plugins/AI/HTN` 下的对应记录位置。

## 开发流程

1. 在开始编写任何功能代码前，必须先讨论需求。
2. 需求确认后，双方都要明确知道要做什么，不能留有歧义。
3. 只有在需求完全清楚后，才能开始写代码。
